## Находка №9 (вскрылась при верификации): reveal обязан падать в «видимо»

`.reveal { opacity: 0 }` в CSS означал: если JS не выполнился или
IntersectionObserver не сработал (скрытая вкладка, старый браузер, сбой
гидрации) — секции остаются НЕВИДИМЫМИ навсегда. Это нарушение
safety/predictability (§16.1): анимация должна деградировать в «контент виден
без анимации», а не в «пустая страница». Переделано: по умолчанию секции
видимы; скрытие (`.will-reveal`) добавляет только сам RevealInit, причём
секции, уже попавшие в первый экран, не анимируются вовсе (контент не должен
«догонять» пользователя). `@media (scripting:none)`-костыль удалён за
ненадобностью.

## Статус

Все 8 пунктов ПРИМЕНЕНЫ этим же коммитом. Скоростные/жестовые вещи из
apple-design (springs, velocity handoff, rubber-banding) на сайте неприменимы —
перетаскиваемых поверхностей нет; если появится мобильный bottom-sheet или
свайп-галерея — брать Motion со spring damping 1.0 / response 0.3–0.4.
